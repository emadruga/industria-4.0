# **Questão 1 - Processo decisório e participação**

| Como as decisões estratégicas e operacionais são tomadas na organização? Qual o nível de participação da equipe no processo decisório? |  |
| :---- | :---- |
| ***Capacidade em medição:*** | Estilo de liderança democrático |

| *Respostas* |  |
| :---: | :---: |
| ***Peso*** | ***Item de resposta*** |
| 0 | Liderança decide sozinha de forma centralizada e autoritária, sem consultar a equipe. Decisões são impostas de cima para baixo. |
| 1 | Liderança toma decisões com base em processos estruturados, mas com mínima ou nenhuma participação da equipe. |
| 2 | Equipe é consultada e pode contribuir com sugestões, mas as decisões finais permanecem centralizadas na liderança. |
| 3 | Decisões são tomadas de forma colaborativa entre liderança e equipe, com consultas formais e análise conjunta de alternativas. |
| 4 | Decisões são co-criadas com a equipe usando análise de dados, avaliação coletiva de alternativas e consenso documentado. |
| 5 | Decisões são tomadas democraticamente com ampla participação, usando análises preditivas e simulações para embasar escolhas estratégicas. |
| 6 | Equipe tem autonomia para tomar decisões de forma descentralizada e adaptativa, com liderança atuando como facilitadora e guardião da visão estratégica. |

| *Possíveis fontes de evidências:* |
| :---- |
| **A) Artefatos e onde buscar** |
| **Decision logs** com registro de participantes, contribuições da equipe, alternativas consideradas e racional da decisão. |
| **Atas de reuniões** (comitês, steering, retrospectivas) mostrando participação multifuncional e processo deliberativo. |
| **Business cases e POCs** desenvolvidos colaborativamente com critérios de sucesso definidos pela equipe. |
| **Registros de votações** ou processos de consenso (ex.: enquetes, matriz de decisão, RACI). |
| **Canais de comunicação** (Slack/Teams) com threads de discussão e co-criação de soluções. |
| **Roadmaps e backlogs** com evidências de priorização colaborativa. |
| **B) Métricas/KPIs** |
| % de decisões com **documentação formal** e evidência de contribuição de múltiplas funções/níveis hierárquicos. |
| **Diversidade de participantes** (número de contribuintes únicos, áreas representadas) em decisões estratégicas. |
| **Lead time de decisão** (do problema identificado até decisão tomada) e taxa de implementação bem-sucedida. |
| **Índice de engajamento** da equipe em processos decisórios (participação em reuniões, contribuições registradas). |
| **Taxa de reversão** de decisões (indicador de qualidade do processo colaborativo). |
| **Net Promoter Score (NPS)** ou pesquisas de satisfação sobre o processo decisório. |
| **C) Sinais por nível** |
| **N0:** decisões unilaterais sem consulta; comunicação top-down; ausência de documentação ou justificativa. |
| **N1:** processos formais existem mas são executados pela liderança isoladamente; equipe apenas recebe comunicados. |
| **N2:** consultas informais ou formais, mas contribuições da equipe têm impacto limitado; decisões já pré-determinadas. |
| **N3:** processo colaborativo estruturado com reuniões de análise conjunta; liderança considera seriamente as contribuições antes de decidir. |
| **N4:** co-criação efetiva com análise de dados compartilhada; consenso buscado e documentado; decisões baseadas em evidências quantitativas. |
| **N5:** processos democráticos maduros com uso de análises preditivas e simulações; equipe vota ou co-decide estratégias com modelos de governança claros. |
| **N6:** autonomia distribuída com células auto-organizadas; liderança facilita e alinha; decisões descentralizadas em tempo real com ajustes adaptativos. |
| **D) Amostragem** |
| Selecionar **5-10 decisões significativas** (estratégicas ou operacionais críticas) dos últimos 90 dias e rastrear: (a) quem participou, (b) como a decisão foi tomada, (c) quais dados foram usados, (d) se houve consenso ou votação, (e) resultado e aprendizados. |

# **Questão 2 - Segurança psicológica para erros e experimentação**

| Como a liderança lida com erros, falhas e riscos assumidos pela equipe? Os colaboradores se sentem confortáveis em admitir erros, fazer perguntas e propor ideias não convencionais? |  |
| :---- | :---- |
| ***Capacidade em medição:*** | Estilo de liderança democrático |

| *Respostas* |  |
| :---: | :---: |
| ***Peso*** | ***Item de resposta*** |
| 0 | Erros são punidos; ambiente de culpabilização e medo. Colaboradores evitam assumir riscos. |
| 1 | Erros são tolerados, mas não discutidos abertamente. Clima de apreensão persiste. |
| 2 | Existe procedimento formal para registro de erros, mas discussão é limitada e não há aprendizado sistemático. |
| 3 | Líderes ocasionalmente admitem erros próprios. Discussões post-mortem pontuais acontecem para incidentes críticos. |
| 4 | Cultura de aprendizado com erros estabelecida. Líderes modelam vulnerabilidade regularmente e incentivam discussão aberta. |
| 5 | Experimentação é encorajada com processos estruturados de gestão de falhas. "Falhar rápido, aprender rápido" é praticado. |
| 6 | Alto nível de segurança psicológica. Erros são celebrados como oportunidades de aprendizado. Inovação floresce sem medo. |

| *Possíveis fontes de evidências:* |
| :---- |
| **A) Artefatos e onde buscar** |
| **Post-mortems e retrospectivas** documentadas com participação ampla da equipe e ações de melhoria identificadas. |
| **Registros de experimentos** e pilotos com hipóteses, resultados (incluindo falhas) e lições aprendidas. |
| **Fóruns de compartilhamento** de erros e aprendizados (ex.: "Failure Fridays", sessões de lições aprendidas). |
| **Repositório de conhecimento** com casos de falhas e como foram resolvidos. |
| **Comunicações da liderança** admitindo erros próprios e modelando vulnerabilidade. |
| **Políticas de experimentação** que explicitam tolerância a falhas em contextos de inovação. |
| **B) Métricas/KPIs** |
| **Índice de segurança psicológica** medido via pesquisas (ex.: escala de Edmondson com 7 itens). |
| % de projetos que incluem **fase de experimentação** com "direito a falhar". |
| **Número de experimentos** executados e taxa de "falhas produtivas" (que geraram aprendizado). |
| **Frequência de reuniões** de retrospectiva e post-mortem (mensal, semanal). |
| % de funcionários que **reportam sentir-se confortáveis** para falar sobre erros sem medo de retaliação. |
| **Taxa de reporte** de near-misses e incidentes (indicador de abertura para falar sobre problemas). |
| **C) Sinais por nível** |
| **N0:** cultura punitiva; erros escondidos; ausência de discussão; alta ansiedade. |
| **N1:** erros tolerados silenciosamente mas não abordados; medo ainda presente. |
| **N2:** processos existem mas são burocráticos; pouco aprendizado real; discussões superficiais. |
| **N3:** alguns líderes compartilham vulnerabilidade; discussões começam a gerar insights; ainda há hesitação. |
| **N4:** aprendizado sistemático; líderes consistentemente modelam abertura; clima de confiança estabelecido. |
| **N5:** experimentação é norma; processos claros para testar e aprender; orçamento dedicado a inovação. |
| **N6:** erros vistos como ativos; celebrações de "falhas inteligentes"; inovação radical acontece naturalmente. |
| **D) Amostragem** |
| Analisar **3-5 incidentes ou falhas recentes**: como foram comunicados, quem participou das discussões, quais ações foram tomadas, se houve punição ou aprendizado, e se mudanças foram implementadas. |

# **Questão 3 - Transparência na comunicação de decisões**

| Como a liderança comunica o contexto, os critérios e o racional por trás das decisões? A equipe tem acesso às informações necessárias para compreender e contribuir efetivamente? |  |
| :---- | :---- |
| ***Capacidade em medição:*** | Estilo de liderança democrático |

| *Respostas* |  |
| :---: | :---: |
| ***Peso*** | ***Item de resposta*** |
| 0 | Decisões tomadas sem comunicação ou justificativa. Equipe não sabe o "porquê" das decisões. |
| 1 | Comunicação unidirecional e tardia. Apenas resultados finais são informados, sem contexto. |
| 2 | Comunicação formal existe através de canais oficiais, mas contexto e critérios são limitados. |
| 3 | Liderança explica o "porquê" das decisões principais. Informações são compartilhadas com algum nível de detalhe. |
| 4 | Compartilhamento proativo de contexto, dados e trade-offs. Comunicação é bidirecional com espaço para perguntas. |
| 5 | Transparência radical. Equipe tem acesso amplo a informações estratégicas, dashboards e análises que embasam decisões. |
| 6 | Cultura de transparência sistêmica. Informações fluem naturalmente; equipe tem visibilidade completa do contexto estratégico e operacional. |

| *Possíveis fontes de evidências:* |
| :---- |
| **A) Artefatos e onde buscar** |
| **Comunicados e anúncios** da liderança explicando não apenas "o quê" mas "por quê" e "como" das decisões. |
| **Dashboards compartilhados** com métricas estratégicas acessíveis a toda equipe. |
| **Atas de reuniões** All-Hands, Town Halls com Q&A registrado e respostas da liderança. |
| **Decision records** públicos ou amplamente acessíveis com racional documentado. |
| **Repositório de informações** estratégicas (intranet, wiki) atualizado regularmente. |
| **Gravações de reuniões** estratégicas disponíveis para equipe que não pôde participar. |
| **B) Métricas/KPIs** |
| % de funcionários que **declaram compreender** o contexto e critérios das decisões estratégicas (survey). |
| **Frequência de comunicação** da liderança sobre estratégia e decisões importantes. |
| **Taxa de participação** em sessões de Q&A e Town Halls. |
| **Número de perguntas respondidas** pela liderança em fóruns abertos. |
| **Tempo médio** entre decisão tomada e comunicação à equipe. |
| **Índice de confiança** na liderança medido via pesquisas de engajamento. |
| **C) Sinais por nível** |
| **N0:** "black box"; decisões surgem sem aviso ou explicação; desconfiança alta. |
| **N1:** comunicação top-down formal mas vazia de contexto; equipe recebe ordens sem entender. |
| **N2:** informações básicas são compartilhadas mas insuficientes para compreensão profunda. |
| **N3:** liderança comunica razões principais; algum espaço para perguntas; transparência parcial. |
| **N4:** contexto rico compartilhado proativamente; dados disponíveis; diálogo bidirecional estabelecido. |
| **N5:** acesso amplo a informações estratégicas; dashboards em tempo real; política de transparência explícita. |
| **N6:** transparência é cultural; informação flui livremente; todos entendem estratégia profundamente. |
| **D) Amostragem** |
| Selecionar **3-5 decisões estratégicas recentes** e verificar: (a) como foram comunicadas, (b) quando foram comunicadas, (c) nível de detalhe do racional, (d) se houve espaço para perguntas, (e) satisfação da equipe com a comunicação. |

# **Questão 4 - Autonomia e delegação de autoridade**

| Como a liderança delega autoridade e promove autonomia da equipe? Os colaboradores têm liberdade e recursos para tomar decisões em suas áreas de responsabilidade? |  |
| :---- | :---- |
| ***Capacidade em medição:*** | Estilo de liderança democrático |

| *Respostas* |  |
| :---: | :---: |
| ***Peso*** | ***Item de resposta*** |
| 0 | Controle total centralizado. Nenhuma autonomia; todas as decisões requerem aprovação da liderança. |
| 1 | Autonomia mínima apenas para tarefas operacionais simples e rotineiras. Liderança mantém controle rígido. |
| 2 | Delegação formal existe, mas múltiplas aprovações são necessárias. Autonomia é mais aparente que real. |
| 3 | Autonomia moderada dentro de limites claramente definidos. Equipe pode decidir "como", mas não "o quê". |
| 4 | Alta autonomia com delegação efetiva. Equipe recebe recursos e suporte adequados para executar decisões. |
| 5 | Empowerment amplo. Equipes decidem sobre recursos, prioridades e métodos com alinhamento estratégico claro. |
| 6 | Autogestão plena. Equipes são autônomas com accountability clara, tomando decisões rapidamente alinhadas à estratégia. |

| *Possíveis fontes de evidências:* |
| :---- |
| **A) Artefatos e onde buscar** |
| **Matriz de delegação** (RACI, DACI) mostrando níveis de autoridade por tipo de decisão. |
| **Políticas de alçadas** documentando quais decisões podem ser tomadas por cada nível. |
| **Exemplos de decisões** tomadas autonomamente pelas equipes sem aprovação prévia da liderança. |
| **Orçamentos descentralizados** geridos pelas próprias equipes. |
| **OKRs/KPIs** definidos pelas equipes (não apenas cascateados de cima). |
| **Estrutura organizacional** (squads, tribos, células) que favorece autonomia. |
| **B) Métricas/KPIs** |
| **Índice de autonomia percebida** medido via pesquisas (escala de autodeterminação). |
| % de decisões tomadas **sem necessidade de aprovação** de níveis superiores. |
| **Tempo médio de aprovação** para decisões que ainda requerem autorização. |
| **Número de aprovadores** médio necessário por tipo de decisão. |
| % do orçamento sob **gestão descentralizada** das equipes. |
| **Taxa de ownership** (colaboradores que se sentem donos de seus projetos/áreas). |
| **C) Sinais por nível** |
| **N0:** microgerenciamento total; todas as decisões sobem; paralisia decisória; frustração alta. |
| **N1:** autonomia apenas para escolhas triviais; controle rígido persiste em tudo relevante. |
| **N2:** processos de delegação existem no papel mas burocracia impede autonomia real. |
| **N3:** autonomia em execução ("como") mas não em definição ("o quê"); limites claros mas restritivos. |
| **N4:** delegação efetiva; equipes decidem com confiança; suporte disponível; poucos bloqueios. |
| **N5:** empowerment real; equipes controlam recursos e prioridades; alinhamento estratégico claro. |
| **N6:** autogestão; decisões rápidas e descentralizadas; liderança facilita e remove impedimentos. |
| **D) Amostragem** |
| Mapear **10-15 decisões recentes** de diferentes tipos e analisar: (a) quem tomou a decisão, (b) quantas aprovações foram necessárias, (c) tempo até decisão, (d) se recursos foram disponibilizados, (e) resultado da decisão. |

| *Glossário* |
| ----- |
| **Estilo de liderança democrático:** Abordagem de liderança participativa na qual os líderes envolvem ativamente os membros da equipe no processo de tomada de decisão, valorizando suas opiniões, conhecimentos e perspectivas. |
| **Decision log:** Documento que registra decisões importantes, incluindo contexto, alternativas consideradas, participantes, critérios utilizados e justificativa da escolha. |
| **Co-criação:** Processo colaborativo onde múltiplos stakeholders contribuem ativamente para criar soluções, estratégias ou decisões, combinando diferentes perspectivas e expertise. |
| **Consenso:** Acordo geral alcançado através de discussão e colaboração, onde todos os participantes apoiam a decisão mesmo que não seja sua primeira escolha. |
| **Análise preditiva:** Uso de dados históricos, algoritmos estatísticos e técnicas de machine learning para prever resultados futuros e embasar decisões estratégicas. |
| **Autonomia distribuída:** Capacidade de equipes e indivíduos tomarem decisões de forma descentralizada dentro de um framework estratégico definido, sem necessidade de aprovação hierárquica constante. |
| **Segurança psicológica:** Crença compartilhada de que o ambiente é seguro para assumir riscos interpessoais, admitir erros, fazer perguntas e expressar ideias sem medo de punição ou humilhação. |
| **Post-mortem:** Reunião ou documento de análise após um evento (sucesso ou falha) para identificar o que funcionou, o que não funcionou e lições aprendidas. |
| **Transparência organizacional:** Grau em que informações sobre decisões, processos e performance são compartilhadas aberta e proativamente com stakeholders relevantes. |
| **Empowerment:** Processo de delegação de poder e autoridade que aumenta a capacidade dos indivíduos de controlar seu trabalho e tomar decisões significativas. |
| **Autogestão:** Modelo organizacional onde equipes têm autoridade para tomar decisões sobre seu trabalho sem aprovação hierárquica constante, dentro de limites estratégicos claros. |
| **RACI/DACI:** Matriz de responsabilidades que define quem é Responsável, Aprovador (Accountable), Consultado e Informado para cada decisão ou tarefa. |
